Pour commencer

Trouvez des données. Combinez-les. Reportez-les sur une carte. Explorez-les. Exportez ce que vous avez créé et partagez-le. Grâce à GEO.CA, trouvez les données qu’il vous faut et utilisez-les de la manière qui fonctionne pour vous. Nos directives étape par étape et nos réponses aux questions les plus fréquentes ci-dessous vous aideront à commencer.

Trouver des données

Commencez directement sur notre Page d’accueil.

La barre de recherche

Utilisez la barre de recherche pour toute la collection de données géoréférencées fédérales, provinciales et territoriales.

Parcourez les thèmes

Ou choisissez l’un de nos neuf thèmes pour consulter toutes les données pertinentes de manière organisée. À partir de chaque page thématique, vous pouvez parcourir tous les ensembles de données appropriés.

Pour vous permettre de commencer rapidement, GEO.CA vous présente aussi des cartes, des applications et des ensembles de données vedettes, sous chaque thème.

Explore GEO.CA’s Discover tab

Découvrir par thèmes, par type de contenu ou par organization

Naviguez jusqu’à l’onglet Découvrir de GEO.ca pour avoir accès à d’autres options de recherche. En plus des Thèmes, vous pouvez faire une recherche par Type de contenu ou Organisation.

Données de base

Vous cherchez des données de base? Cliquez sur Données de base du Canada afin de trouver celles qui décrivent les infrastructures hydrauliques, terrestres et bâties et les fondements géographiques essentiels pour soutenir et contextualiser des données thématiques.

Recherche avancée

Pour des recherches plus poussées, GEO.ca offre trois autres options : Recherche, Mot clé uniquement et Filtres. Vous pouvez vous servir de ces trois méthodes, seules ou combinées.

L’application de Recherche

L’application de Recherche utilise une carte pour chercher des données dans le secteur indiqué. Faites un panoramique et un zoom sur la carte pour filtrer le contenu selon votre secteur d’intérêt. Les résultats sont rafraîchis en fonction du secteur de la carte

Recherches par mots clés

Utilisez des mots clés pour faire des recherches par sujet plutôt que par emplacement. Tapez les mots clés liés aux données que vous cherchez dans la barre de recherche. L’empreinte géographique de chaque résultat de recherche s’affiche également.

Filtres

Choisissez un filtre avancé pour préciser les résultats de recherche à partir des données sur l’organisation, le type, le thème et des données de base. Ajoutez ou supprimez des filtres pour obtenir les résultats qu’il vous faut.

Consulter des données

« Voir enregistrement »

Pour consulter les données après avoir fait une recherche, cliquez sur « Voir enregistrement », où vous pouvez lire l’information sur l’ensemble de données.

« Voir empreinte  »

Quand vous faites une recherche dans Recherche, vous pouvez cliquer sur « Voir empreinte » pour voir l’étendue spatiale de l’ensemble de données sélectionné.

La page «Voir enregistrement »

À partir de la page «Voir enregistrement », vous pouvez choisir de consulter les données et d’interagir avec ces données sous forme de carte ou avoir accès à un enregistrement de métadonnées détaillé.

Créer une carte

Suivez ces étapes simples pour sélectionner et combiner du contenu et créer votre propre carte à analyser et à partager :

Ajoutez des données de GEO.ca

Pour ajouter des données à votre carte, cliquez sur « Ajouter à Ma carte » dans la section de l’enregistrement des données. Cliquez sur « Ma carte » dans le coin supérieur droit pour vous rendre à votre carte.

Téléversez des données d’autres sources

Pour ajouter des données provenant d’autres sources, vous pouvez les importer à l’aide de l’application cartographique. Vous pouvez ajouter des données ou des services en téléversant un fichier ou à partir d’une adresse URL sur le Web.

Supprimez des données de votre carte

Pour supprimer des données de votre carte, cliquez sur « Ma carte » dans le coin supérieur droit. Supprimez la couche en cliquant sur le « X » situé à côté de son nom.

Exportez votre carte

Pour exporter votre carte, élargissez le menu dans le panneau de la couche de la carte et cliquez sur « Exporter », ce qui vous permet de télécharger votre carte.

Q&As – Questions & Answers

GEO.CA is an online environment where people can discover, visualize, and analyze open Canadian geospatial data and information! For more information, please visit About Us.

The geospatial data that can be accessed on GEO.ca comes from over 21 federal departments, as well as provincial, and territorial contributors. For more information, please visit Contributors.

The owner of each dataset is responsible for creating, contributing, and updating their data. Information on data publications, respective update frequency, and data ownership can be found on the dataset’s metadata page.

Please contact us , or the data owner listed on the dataset’s page, and we will try to solve the issue as soon as possible.

When satisfied with the data displayed in the Map Viewer, you can download and print the maps to obtain a physical copy.

Web Services transmit information, such as map images, over standardized internet protocols. Users can receive specific information by creating an HTTP request. Some types of web services found on GEO.CA include Web Map Service (WMS) and ArcGIS REST.

APIs are internet services that will provide a subset of data and information when a specific web request is made by the user. Discover our available APIs.

Applications are software designed for end-users to accomplish specific tasks. Applications can make use of APIs, datasets, collections, and web services.

A dataset is a single, packaged set of information. It consists of multiple features (or records) and their attributes. In terms of a geographic dataset, each record will have associated geographic information (i.e., geographic coordinates).

A set of datasets that have been grouped together for a specific purpose.

Data Formats

Although there are many different data formats available, these are some of the most common ones you will come across on GEO.CA

A GDB is a geodatabase, an ESRI data format. There can be Personal Geodatabases or File Geodatabases (FGDB). They are a way to store geographic information in a single file. You can store, query, and manage files in geodatabases. Although geodatabases are an ESRI data format, most other software are now able to access them.

KML/KMZ (Keyhole Markup Language) is the data format for geographic information used by Google. They are easy to use by simply dragging and dropping them into Google Earth Pro to view.

Most GIS software has capabilities to import KML/KMZ files or transform them to other formats (watch out for your attribute tables! They can often become impractical when importing to other software).

They are both image files, like jpeg or png, but a geotiff has geographic information imbedded into it. You can view a geotiff in a map and it will be in the proper location.

A shapefile (SHP) file is an ESRI specific data format. It has become so widely used that nearly all other software can read it easily. Shapefiles store geographic features like vectors (i.e. points, lines, and polygons) and their attributes.

GPKGs are an open data format for geographic information, built as an SQLite database container. It supports both raster and vector data. A geopackage is provided as a single file containing data and metadata tables.

Contributors

Contributors are federal, provincial and territorial organisations that work GEO.CA to geo-enable their data to be made available on our site. Find a list of all our contributors.

You can become a contributor if you are a Government of Canada public servant or member of a Provincial or Territorial organization. For more information on becoming a federal contributor, please visit our Contributors page. For more information on becoming a Provincial or Territorial contributor, please contact us.

Contact us if you require assistance with publishing metadata.

The Harmonized North American Profile is the metadata standard used by GEO.CA contributors to ensure quality and facilitate the collection, sharing and use of authentic and accurate geospatial content. It is based on the North American Profile of ISO 19115:2003 (NAP) minimum mandatory metadata elements.
